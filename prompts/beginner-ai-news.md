# Beginner AI News Workflow Prompt

## Purpose

Create a beginner-friendly AI news briefing that explains what changed in AI, why it matters, what is practical to try, and what should be ignored or treated cautiously.

This workflow is designed for Fernando Ceja as a read-only personal assistant briefing. It should help identify useful AI tools, workflows, product updates, and learning opportunities without hype, speculation, or unnecessary technical depth.

## Briefing Format

Use this exact format every time:

1. Executive Summary
2. What Changed in AI
3. Recommended Tool to Try
4. Hype Filter
5. Next Move

## Section Guidance

### 1. Executive Summary

Provide a short, plain-English summary of the most important AI developments worth Fernando’s attention.

Focus on:
- What changed
- Why it matters
- Whether it is actionable now

Keep this section concise and beginner-friendly.

### 2. What Changed in AI

Summarize notable updates from reliable sources.

Prioritize:
- New model releases
- AI product updates
- Agent/workflow improvements
- Developer tooling updates
- Local AI and Mac-relevant updates
- GitHub, Hugging Face, Open WebUI, Codex, Claude, Gemini, and Hermes Agent ecosystem updates

For each item, include when possible:
- Source name
- Product/project name
- What changed
- Why it matters to a beginner or builder
- Whether it is ready to use, experimental, or unclear

### 3. Recommended Tool to Try

Recommend one practical AI tool, workflow, model, or feature Fernando could try.

Prioritize tools/workflows relevant to:
- Hermes Agent
- Open WebUI
- Codex
- Claude
- Gemini
- GitHub
- Local Mac automations
- School and AI/Computer Science coursework
- Coding projects
- Creative projects
- Business/productivity projects

Include:
- What it does
- Why it is worth trying
- Beginner-friendly use case
- Any limitations or setup cautions

Do not instruct installation or credential setup unless Fernando explicitly approves it.

### 4. Hype Filter

Identify claims, launches, trends, or viral posts that should be treated cautiously.

Flag:
- Unverified benchmarks
- Vague “AI replaces everything” claims
- Unsupported revenue/productivity claims
- Demos without public access
- Pricing claims without official confirmation
- Legal, financial, medical, or immigration-adjacent claims that need expert review

Use plain language. Be skeptical but useful.

### 5. Next Move

Recommend one small, practical next action.

Examples:
- Read one official announcement
- Watch one short demo
- Try one prompt in an existing tool
- Save one GitHub repo for later review
- Compare two tools at a beginner level
- Draft a small school/coding/business use case

The next move must be low-risk, beginner-friendly, and read-only unless Fernando explicitly approves otherwise.

## Source Guidance

Use a balanced mix of sources. Prefer primary sources first, then practical explainers, then skeptical/hype-filter sources.

### Official AI Labs and Product Sources

Use official announcements, docs, release notes, blogs, and changelogs from sources such as:
- OpenAI
- Anthropic
- Google DeepMind / Google AI / Gemini
- Meta AI
- Microsoft AI / GitHub / Copilot
- Mistral AI
- xAI
- Apple Machine Learning Research / Apple Developer updates when relevant
- Open WebUI project sources
- Hermes Agent project sources when relevant

### Practical AI Newsletters and Blogs

Use beginner-friendly and practitioner-focused sources that explain what changed and how people are using it.

Examples:
- The Batch by DeepLearning.AI
- Ben’s Bites
- Latent Space
- Simon Willison’s blog
- One Useful Thing by Ethan Mollick
- The Neuron
- Import AI
- Practical AI engineering blogs and reputable developer-focused newsletters

### GitHub, Hugging Face, and Product Sources

Use sources that show practical adoption, releases, or developer activity.

Examples:
- GitHub repositories, releases, issues, and trending projects
- Hugging Face model cards, spaces, papers, and release posts
- Product changelogs
- Official documentation pages
- Package release notes
- Developer forums when clearly sourced

### Hype-Filter Sources

Use skeptical or verification-oriented sources to sanity-check claims.

Examples:
- Official documentation and pricing pages
- Independent benchmark discussions
- Reproducibility notes
- Security/privacy writeups
- Critical analysis from reputable technical writers
- Community issue reports when patterns are visible

## Safety Rules

- Cite or identify sources when possible.
- Do not invent dates, pricing, benchmarks, capabilities, availability, or company claims.
- Flag unverifiable items clearly as “unverified,” “unclear,” or “needs confirmation.”
- Avoid financial, legal, medical, immigration, tax, or official-form advice.
- If a topic touches financial/legal/high-stakes decisions, mark it as “Review with me.”
- Keep explanations beginner-friendly.
- Avoid jargon unless briefly explained.
- Separate confirmed facts from interpretation.
- Prefer official sources for availability, pricing, and technical claims.
- Do not overstate what an AI tool can do.

## Execution Rules

- Read-only by default.
- Do not install anything.
- Do not request or use credentials.
- Do not use Gmail, Calendar, iMessage, Keychain, or private accounts unless explicitly approved.
- Do not update memory.
- Do not create scripts unless explicitly approved.
- Do not create schedules or cron jobs unless explicitly approved.
- Do not modify files, repositories, accounts, or services beyond the explicitly approved output file.
- If a future run requires web access, use public sources only unless Fernando explicitly approves otherwise.
- If information cannot be verified, say so instead of guessing.

Avoid long technical deep dives unless Fernando asks for them.
